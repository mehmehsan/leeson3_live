Lesson-02 : Program Structure

Expression and statement

  • a piece of code is expression
  • something that changes state of machine is statement , these are side effects

    • Bindings

  • to catch and hold values, js uses variables/bindings
  • keyword << names << operator(=) << bindings[KNOB]
  • now use name as an expression
  • if bindings don't have any value they have nothing to catch, they get undefined values
  • var used for declaring variables
  • const for declaring constants

    • Binding names

  • words with special meanings are called keywords
  • don't try to define reserved keywords, bcz it will not happen

    The environment

  • Collection of binding and values is called environment

    Functions

  • values provided in default environment is called function
  • exapmle prompt("Enter name"), is called involking, calling, or applying.
  • values given to functions are called arguments

    Console.log() function

  • function that takes some text and revels it in output device

    Return values

  • displaying values and alerts is a "side effect"

    Control flow

  • statements gets executed from top to bottom

    Conditional execution

  • create branching using conditional execution
  • {} braces used to group any statement into "block"
  • else if is used to chain mutliple branches
  • for mutiple execution we use loop
  • loop is executed if coonditional statement holds true value

    for loop structure

  • for ( initialize value, conditional statement, updates iteration){ }
  • if conditional statement gets false, the loop breaks out, if false statement doesn't come it goes to infinite.

    Updating bindings succinctly

  • it includes increment/ decrement like counter = counter + 1; similar shortcut counter + = 1; even shorter counter ++; ++ counter
